# Інтелектуальна система управління особистими фінансами з прогнозуванням витрат

---

## 1. Загальна інформація про проєкт

**Інтелектуальна система управління особистими фінансами з прогнозуванням витрат** — це настільний програмний застосунок, розроблений мовою **Python**, який призначений для:

- ведення обліку доходів і витрат;
- автоматичного розрахунку балансу;
- аналізу фінансової активності користувача;
- візуалізації даних у вигляді графіків;
- прогнозування майбутніх витрат;
- роботи з кількома користувацькими профілями;
- експорту транзакцій у файл формату CSV.

Програма реалізована у вигляді графічного інтерфейсу користувача (**GUI**) із використанням бібліотеки **CustomTkinter**.

---

## 2. Призначення системи

Метою програми є створення зручного локального інструмента для персонального фінансового обліку, який дозволяє не лише зберігати дані про доходи та витрати, але й аналізувати їх, знаходити тенденції та отримувати прогноз щодо майбутніх витрат на основі історичних записів.

Система може використовуватись:

- для персонального контролю бюджету;
- для демонстрації можливостей аналітичних та прогнозних модулів;
- як навчальний програмний продукт у межах бакалаврської кваліфікаційної роботи.

---

## 3. Основні можливості програми

### 3.1. Облік доходів

Доходи — це всі грошові надходження користувача, наприклад:

- зарплата;
- премія;
- подарунок;
- підробіток;
- фриланс;
- повернення боргу;
- бонус.

Для кожного доходу користувач вводить:

- тип операції: **Дохід**;
- категорію;
- суму;
- дату у форматі `YYYY-MM-DD`.

---

### 3.2. Облік витрат

Витрати — це всі фінансові витрати користувача, наприклад:

- продукти;
- транспорт;
- комунальні;
- догляд;
- аптека;
- розваги;
- кафе;
- техніка;
- одяг;
- подарунки.

Для кожної витрати користувач вводить:

- тип операції: **Витрата**;
- категорію;
- суму;
- дату у форматі `YYYY-MM-DD`.

---

### 3.3. Автоматичний розрахунок балансу

Баланс обчислюється автоматично за формулою:

**Баланс = Загальна сума доходів − Загальна сума витрат**

Цей показник відображається на головній сторінці програми.

---

### 3.4. Робота з кількома профілями

У системі реалізовано механізм роботи з кількома користувацькими профілями.

Користувач може:

- створити новий профіль;
- перемикатися між профілями;
- перейменовувати профіль;
- видаляти профіль;
- переглядати коротку інформацію про активний профіль:
  - кількість доходів;
  - кількість витрат;
  - поточний баланс.

Кожен профіль має власний незалежний набір фінансових даних.

---

### 3.5. Перегляд, редагування та видалення транзакцій

У розділі **«Транзакції»** користувач може:

- переглядати всі транзакції поточного профілю;
- оновити список;
- редагувати окрему транзакцію;
- видаляти транзакцію.

Під час редагування можна змінити:

- тип транзакції;
- суму;
- категорію;
- дату.

---

### 3.6. Аналітика доходів і витрат

Система автоматично виконує фінансовий аналіз і надає:

- загальну суму доходів;
- загальну суму витрат;
- баланс;
- середні витрати;
- найбільші категорії витрат;
- текстові рекомендації на основі фінансової активності.

---

### 3.7. Побудова графіків

Програма будує такі графіки:

- кругова діаграма витрат по категоріях;
- кругова діаграма доходів по категоріях;
- лінійний графік динаміки витрат по місяцях;
- графік прогнозування витрат.

Графіки побудовані за допомогою бібліотеки **Matplotlib**.

---

### 3.8. Прогнозування витрат

У межах теми бакалаврської роботи в систему інтегровано модуль прогнозування витрат.

Користувач може обрати:

#### Вікно аналізу:
- 30 днів;
- 60 днів;
- 90 днів.

#### Модель прогнозування:
- `linear` — лінійна регресія (стабілізована);
- `average` — середнє значення;
- `weighted` — зважене середнє.

Система формує:

- прогноз загальних витрат на наступні 30 днів;
- значення метрик якості:
  - **MAE**
  - **MSE**
  - **R²**
- словесну оцінку тренду витрат;
- прогноз по категоріях;
- текстові AI-рекомендації;
- графік історичних даних та прогнозу.

---

### 3.9. Експорт у CSV

Користувач може експортувати транзакції активного профілю у файл `CSV`.

Це дає можливість:

- відкрити дані в Excel;
- опрацювати їх у Google Sheets;
- зберегти резервну копію;
- використовувати файл для подальшого аналізу.

## 5. Формат локального сховища даних

Програма використовує файл data/data.json як локальну JSON-базу даних.

Приклад структури:
```bash
{
    "active_profile": "Основний",
    "profiles": {
        "Основний": {
            "income": [
                {
                    "amount": 12000,
                    "category": "Зарплата",
                    "date": "2026-04-01"
                }
            ],
            "expenses": [
                {
                    "amount": 350,
                    "category": "Продукти",
                    "date": "2026-04-02"
                }
            ]
        },
        "Профіль 2": {
            "income": [],
            "expenses": []
        }
    }
}
```
Пояснення структури:
active_profile — назва активного профілю;
profiles — словник усіх профілів;
income — список доходів вибраного профілю;
expenses — список витрат вибраного профілю.

## 8. Правила введення даних
## 8.1. Формат дати
Дата повинна вводитися тільки у форматі:
YYYY-MM-DD
Приклад:
2026-05-30
Якщо дата введена некоректно, програма повідомляє про помилку.
---
## 8.2. Формат суми
Сума повинна бути:
числом;
більшою за 0.
Приклади коректного введення:
250
1000.50
3400

## 10. Особливості реалізації прогнозування
Для прогнозування в системі використано підхід, який дозволяє працювати з нерівномірними фінансовими рядами.
Оскільки витрати реального користувача можуть суттєво коливатися від дня до дня, модуль прогнозування підтримує кілька моделей та ковзні часові вікна аналізу. Це дозволяє зробити прогноз гнучкішим та краще адаптувати його до різних сценаріїв використання.
